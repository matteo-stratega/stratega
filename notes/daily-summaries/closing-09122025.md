## Session 2 - PDF/Image Compression (sera)

### Obiettivo
Ridurre dimensioni PDF e immagini per compatibilità con Claude API.

### Completato

#### 1. Compressione Immagini
**Cartella:** `Photos/` → `Photos-compressed/`
- **Tool:** ImageMagick (installato via Homebrew)
- **Impostazioni:** resize 1920px max, quality 70%
- **Risultato:** 468 MB → 115 MB (75% riduzione, 85 immagini)

#### 2. Split PDF (soluzione per API error 400)
**Cartella:** `Catalogues-split/`
- **Tool:** Ghostscript con page range extraction
- **Strategia:** Split in parti da ~25 pagine per mantenere qualità

| File | Dimensione | Pagine |
|------|------------|--------|
| SINGLE_BEDS_part1-4 | 25 MB totale | 97 pag |
| BEDS_part1-6 | 58 MB totale | 151 pag |

#### 3. Estrazione Testo
**Cartella:** `Catalogues-text/`
- **Tool:** pdftotext (poppler)
- **Risultato:** ~279 KB totale in file .txt

### Tools Installati (Homebrew)
- ImageMagick (`magick`)
- Ghostscript (`gs`)
- Poppler (`pdftotext`, `pdfinfo`)

### Note Tecniche
- `pdfseparate` di poppler crea file enormi - non usare per split
- Ghostscript con `-dFirstPage` e `-dLastPage` mantiene efficienza

### Struttura Finale

### 2. Folder Reorganization - Root Cause Analysis

**Problema:** Terzo tentativo di riorganizzare le cartelle Stratega

**Finding Principale:**
- Il repo git (`/AI-Projects/stratega/`) è relativamente pulito
- Il vero casino è in Google Drive (`/Drive Stratega/`) - **439 file nella root**

**8 Root Causes Identificati:**
1. Due sistemi paralleli (git vs Drive) - confusione source of truth
2. Dichiarazioni di vittoria false (Round 6 = "100%" → ammesso misleading)
3. Nessun sistema di prevenzione
4. No enforcement (cartelle esistono ma non usate)
5. Duplicati mai risolti (138 file da gennaio)
6. Naming convention senza enforcement
7. Cleanup 100% manuale
8. Trattato come progetto, non sistema operativo

**Pattern:** Cleanup → Vittoria dichiarata → 7-10 giorni → Problema ritorna
