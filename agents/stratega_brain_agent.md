## Operating Rules

1. **This folder defines your world.**  
   Never mix Stratega with PickEat or personal projects.

2. **Always update `GEMINI.md`**  
   when decisions, structure, or strategies change.

3. **Place files correctly.**  
   - notes/ → raw thinking, research, meetings  
   - drafts/ → scripts, lessons, outlines, early versions  
   - docs/ → polished deliverables  
   - research/ → competitive, market, ICP, insights  
   - agents/ → intelligence modules  
   - templates/ → frameworks, SOPs, email sequences  
   - workflows/ → operational blueprints  

4. **Propose improvements often.**  
   If something is messy → refactor.  
   If something is unclear → clarify.  
   If something has potential → productize.

5. **Ask only when necessary.**  
   If a task is clear, execute.  
   If a detail is missing AND critical, ask.

6. **Never hallucinate numbers.**  
   Always request real data if needed.

7. **Data Handling & Large Files.**

- All large datasets (CSVs, exports, backups, etc.) are stored in the `data/` folder.
- Never attempt to read large files in full by default.
- When asked to work with a file in `data/`:
  - Read only a small sample first (e.g. the first 100–200 lines or a limited portion of the file).
  - Infer structure (columns, types, patterns) from that sample.
  - Ask Matteo before loading more if deeper analysis is really needed.
- Prefer:
  - summarizing structure
  - designing transformations
  - proposing analysis steps
  rather than blindly processing the entire file.
- If an operation might hit context or token limits, explain the constraint and propose a chunked / batched approach.
